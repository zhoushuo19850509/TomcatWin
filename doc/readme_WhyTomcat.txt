总结一下为啥要学习Tomcat

1.从宏观上来说，分为科研和应用两大类
因为种种原因，主要是因为资质的原因，
绝大部分人可能无法投身科研，只能写应用代码。
从应用层来说，Tomcat可以说是一个非常优秀的开源产品了。
所以学习Tomcat源码可以在应用层框架内，使得自己获得较大的提升。

2.能够夯实我们的java基础
以tomcat学习为契机，深入了解Java各个主题，包括：多线程开发、反射等。

3.把tomcat架构应用到实际应用中
比如，我们要实现一个自己的数据库，叫做SimpleDB。
那么客户端(JDBC)和服务端(SimpleDB)怎么交互呢？
这时，我们就能通过tomcat的设计思想，设计数据库服务端的通讯架构。
因为我们可以设想一下，数据库端接收到来自客户端的SQL请求，
和tomcat接收到servlet请求有啥区别呢？
无非就是那么几个步骤：
解析SQL
判断SQL类型
如果是select请求，那么就从Local data file中找出所需的数据
如果是insert请求，那么就把新的数据插入到local data file中
如果是update请求，那么更新local data file的数据
然后记录一下日志。
......
虽然数据库的实现远远要比这个tomcat执行servlet类要复杂，
但是大致架构都是相通的。

4.设计指导
我们在软件设计的时候，往往会面临如下的情况：
没有设计思路；有设计思路，但是需要在不同的设计方案之间作抉择
这时我们可以从tomcat中汲取设计思路，作为我们设计的参考。
你可以说：tomcat在XX模块中是这么设计的，这样的设计方式有XXX这些优点，经受了实践的考验。
